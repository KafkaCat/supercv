# WonderCV 功能测试用例

## 1. 导入确认流程测试

### 1.1 正常导入测试
**前置条件**: 准备一个标准的 PDF 简历（如根目录下的 `软件工程师-李国嵩.pdf`）。
**操作步骤**:
1. 点击工具栏的“导入”按钮。
2. 选择 PDF 文件。
3. **预期结果**: 
    - 不会直接覆盖当前简历。
    - 弹出一个“确认导入内容”的 Modal 对话框。
    - 对话框中显示提取到的姓名、电话、邮箱、技能等信息。
4. **验证动作**:
    - 修改对话框中的“姓名”字段（例如在名字后加“测试”）。
    - 点击“确认并导入”。
    - **预期结果**: 页面编辑器中的姓名变为修改后的内容，其他信息也正确填入。

### 1.2 导入失败/手动提取测试
**前置条件**: 准备一个纯图片扫描的 PDF 或内容极少的 PDF。
**操作步骤**:
1. 点击“导入”并选择该文件。
2. **预期结果**: 
    - 系统提示“提取的文本内容过少...是否尝试手动粘贴？”。
3. **验证动作**:
    - 点击“确定”。
    - 在弹出的文本框中粘贴一段简历文本（如：`张三 13800138000 test@test.com`）。
    - 点击“解析并填入”。
    - **预期结果**: 简历信息正确更新为张三的信息。

## 2. 智能一页排版测试

### 2.1 内容过少时
**前置条件**: 新建一个简历，只填写少量信息。
**操作步骤**:
1. 点击工具栏紫色的“智能一页”按钮。
2. **预期结果**: 
    - 字体大小应变大（如 14px），行间距变大（1.5），边距变大（15mm），使内容看起来更充实。

### 2.2 内容过多时
**前置条件**: 填写大量工作经历和教育经历，使预览超过一页。
**操作步骤**:
1. 点击“智能一页”按钮（可能需要点击两次来切换模式，因为目前实现是简单的 toggle）。
2. **预期结果**: 
    - 字体大小应变小（如 12px），行间距变小（1.3），边距变小（10mm），尝试将内容压缩进一页。

## 3. 样式检查
**操作步骤**:
1. 观察预览区域的标题（如“个人总结”、“教育经历”）。
2. **预期结果**: 标题颜色应为**黑色**（#111827 或类似），而非之前的深蓝色。
